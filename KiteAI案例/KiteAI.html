创始背景与战略转型

KiteAI最初以Zettablock的身份出现，成立于2022年，专注于Web3应用中的大规模实时数据基础设施。然而，随着AI Agent概念的兴起，团队敏锐地察觉到一个全新的、万亿级的市场机会——AI Agent自主经济。

在2024年11月，KiteAI推出了创新产品，并完成了战略转型，专注于为AI Agent构建Layer 1支付区块链。这一转变体现了创始团队对未来趋势的深刻洞察：传统的人类中心化支付系统无法满足AI Agent对身份、信任、可编程治理和微支付的独特需求。

核心领导团队拥有深厚的AI、大数据和区块链背景。CEO Chi Zhang曾是Databricks产品管理负责人、加州大学伯克利分校AI博士；CTO Scott Shi曾在Uber负责实时AI基础设施，并是Salesforce Einstein AI的创始工程师。这种跨学科的复合背景使团队能够从第一性原理出发，为AI Agent自主经济设计基础设施。

市场定位与业务模式

KiteAI的市场定位是"AI Agent支付领域的Layer 1基础设施"，这一定位既独特又前瞻性。它不是传统的支付公司，也不是通用的区块链平台，而是一个专门为AI Agent设计的操作系统级别的基础设施。

其业务模式是纯粹的ToB（企业对企业）。核心客户包括：(1) AI Agent开发者和初创团队，(2) 寻求AI自动化的大型企业，(3) 数据和API服务提供商，(4) Web3项目和区块链开发者。

收入来源预计包含多个维度：链上交易费用（AI Agent每笔微支付的极低Gas费），Agent App Store佣金（平台内服务提供商的分成），企业级SaaS订阅（高级功能和管理工具），以及开发者API使用费。这种多元化的收入结构既能保证底层基础设施的广泛采用，又能通过增值服务实现可观的商业化。

产品矩阵概览

KiteAI的核心产品矩阵围绕其作为"AI Agent Layer 1支付区块链"的定位展开，形成了一个紧密集成的生态系统：

1. Kite Layer 1区块链
这是整个生态的基础。一个专为AI Agent优化的区块链网络，具有极高的吞吐量和极低的延迟。平均出块时间仅1秒，日交互量可超百万，单笔交易费用低于$0.000001。这种性能指标使得AI Agent能够进行真正意义上的高频、微额交易。

2. Agent Passport（身份系统）
为每个AI Agent颁发可验证的加密身份。这个"护照"记录了Agent的权限、归属方、操作范围等关键信息。这是整个生态的信任基础，解决了"AI Agent是谁"和"它被允许做什么"的问题。

3. Agent App Store（服务市场）
一个供AI Agent发现和访问各种服务（API、数据、工具）的平台。商家和开发者可以在此发布服务，接受AI Agent的微支付。这个市场形成了双边网络效应——越多的服务吸引越多的Agent，越多的Agent驱动更多的服务商入驻。

4. 可编程治理（Programmable Governance）
通过智能合约为AI Agent设定精细化的行为规则、权限范围和支出限制。这些规则在协议层面强制执行，确保AI Agent在自主运行时不会越权。

使用场景一：AI购物Agent

场景描述： 用户授权一个AI购物Agent在电商平台（如Shopify）上自主寻找和购买特定商品。

传统方案的痛点： Agent需要通过复杂的OAuth握手和API密钥管理来完成认证，效率低下且存在安全风险。

KiteAI解决方案：

  • AI购物Agent获得Agent Passport，拥有可验证的加密身份
  • 在Kite Agent App Store中发现Shopify商户的服务列表
  • 当找到目标商品时，Agent通过KiteAI自动与商户协商价格
  • 使用稳定币进行即时、链上结算，整个过程可追溯且符合预设权限
  • Shopify商户可通过App Store选择加入，扩大销售渠道

效果： 购物效率提升100倍，交易成本降低99%，同时完全透明和可审计。

使用场景二：供应链自动化

场景描述： 一家制造企业拥有多个AI Agent分别负责采购、生产调度和物流，需要进行频繁交易和结算。

传统方案的痛点： 人工审批周期长、对账复杂、结算缓慢，容易出错且难以审计。

KiteAI解决方案：

  • 采购Agent、生产Agent、物流Agent各拥有独立的Agent Passport
  • 采购Agent自主在Agent App Store中找到合适供应商，完成谈判
  • 原材料到达后，生产Agent验证质量并通过智能合约触发尾款支付
  • 物流完成后，物流Agent自动向运输公司支付运费
  • 企业管理层可实时查看所有Agent的交易活动、成本和审计记录

效果： 支付周期从周级缩短到秒级，人工成本降低80%，供应链透明度大幅提升。

核心技术架构

KiteAI的技术架构是其核心竞争力所在。它并非简单地调用现有API或区块链，而是从底层重新设计了一个专为AI Agent优化的系统。

三层身份架构：

  • 用户层(User)： 根权限方，代表人类用户或组织
  • Agent层： 获得用户委托的AI实体，拥有独立Agent Passport
  • 会话层(Session)： 临时、细粒度的权限，用于特定任务

这种分层设计通过BIP-32分层确定性密钥派生实现，确保权限的数学委托和安全管理。

Proof of Artificial Intelligence (PoAI)共识机制： 这是KiteAI的核心创新。不同于比特币的PoW（工作量证明）或以太坊的PoS（权益证明），PoAI旨在激励参与者为AI Agent生态贡献有意义的工作——验证Agent身份、确保交易合法性、数据真实性等。这个共识机制体现了"公平和透明的协作"理念。

AI能力与差异化优势

KiteAI的AI能力并非体现在生成式AI的输出质量上，而是体现在其作为"AI Agent操作系统"的底层支撑能力：

1. AI Agent身份认证与管理
为每个AI Agent提供唯一的可验证加密身份，解决了"AI Agent是什么"的根本问题。

2. 可编程治理与行为约束
允许用户通过智能合约为AI Agent设定精细化的行为规则、权限范围和支出限制，确保其在自主运行时的可控性。

3. Agent原生微支付
提供高效率、低成本的稳定币支付通道，支持AI Agent之间的即时、高频交易。

4. 合规性与审计追踪
所有交易和关键行为都在链上留下不可篡改的记录，满足欧盟AI Act等监管要求。

5. 互操作性与生态集成
与x402协议、Google A2A、Anthropic MCP、OAuth 2.1等主流Agent协议原生兼容，成为通用执行层而非孤立生态。

技术壁垒与风险

技术壁垒：

PoAI共识机制的独特性： 新型共识算法难以被复制，形成专有技术优势。
三层身份+可编程治理的复杂性： 设计和维护难度高，需要深厚的区块链和AI领域知识。
高性能微支付基础设施： 1秒出块和接近零的交易费用，通过状态通道等Layer 2技术实现，技术难度大。

技术风险：

1. PoAI共识的成熟度： 作为新型共识机制，其长期运行的稳定性和安全性仍需市场验证。

2. 智能合约安全： 智能合约漏洞是区块链领域的常见风险，需要持续的代码审计。

3. 互操作性挑战： 保持与快速演进的多种Agent协议的兼容性，是持续的技术挑战。

4. 性能扩展性： 随着AI Agent数量指数级增长，网络可能面临性能瓶颈。

AI Agent经济时代的基础设施定位

KiteAI最核心的战略洞察是：AI Agent经济的爆发不是遥远的未来，而是正在眼前发生。而在这个新经济中，支付和身份认证是最基础的需求。

KiteAI的洞察在于：传统支付系统（Visa、PayPal）以人类为中心，无法处理AI Agent的高频、微额、需要可验证身份的交易；通用区块链（以太坊、Solana）缺乏AI Agent原生的身份和治理能力。因此，一个"为AI Agent而生"的Layer 1区块链是必然的。

支付基础设施的演进总是领先于应用的爆发——KiteAI正在为AI Agent大爆发提前铺设轨道。

AI与区块链的融合点

KiteAI代表了AI与区块链融合的一个重要方向。这种融合并不是简单的"AI + 区块链"，而是找到了它们各自的最佳应用场景的交汇点。

区块链的核心价值是提供"不可篡改的记录"和"去中心化的信任"。在AI Agent的语境下，这些价值得到了升华：(1) Agent身份的不可篡改记录，确保了身份的唯一性和可追溯性，(2) 链上的不可篡改交易记录，满足了企业对AI Agent行为的审计需求，(3) 智能合约提供的可编程治理，让企业能够以代码的形式强制执行对AI Agent的约束。

反过来说，AI并未替代区块链，而是为其找到了最合适的应用场景。这种相互成就的关系，往往预示着一个新的、巨大的市场。

真正的AI × Web3融合应该是找到它们各自最擅长的东西，然后让两者相互赋能。KiteAI做到了这一点。
